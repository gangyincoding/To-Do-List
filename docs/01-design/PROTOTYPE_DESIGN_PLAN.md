# TaskAI 产品原型设计计划

## 🎯 设计目标
根据PRODUCT.MD第4步要求，设计TaskAI移动端产品的页面布局和交互逻辑（非视觉设计）。

## 📱 第4步：做产品原型图

### 🎨 核心设计原则

#### 1. 语音优先原则
- 以语音输入为主要交互方式
- 录音界面必须简洁直观
- 语音反馈机制要即时明确

#### 2. AI智能展示原则
- AI拆分结果要清晰展示
- 体现AI的智能价值
- 用户可以理解和编辑AI结果

#### 3. 移动端友好原则
- 支持单手操作
- 界面元素尺寸适合移动端
- 交互流程简单高效

## 📋 页面结构设计

### 1. 主页面（Home Page）

#### 页面功能
- 语音输入入口（核心功能）
- 今日任务快速预览
- 快速操作按钮

#### 页面布局
```
┌─────────────────────────┐
│ 🎤 TaskAI                │  ← 页面标题
├─────────────────────────┤
│                         │
│     🎤 录音按钮           │  ← 核心交互元素
│   （点击或长按录音）        │
│                         │
│    "点击开始录音"         │  ← 状态提示
│                         │
├─────────────────────────┤
│ 今日任务 (3)              │  ← 今日任务预览
│ ┌─────────────────────┐   │
│ │ ☐ 完成项目评审会议    │   │
│ │ ☐ 准备PPT和数据      │   │
│ │ ☐ 联系参会人员        │   │
│ └─────────────────────┘   │
├─────────────────────────┤
│ [+] 添加任务  [设置] ⚙️    │  ← 快速操作
└─────────────────────────┘
```

#### 交互逻辑
1. **进入页面**：显示今日任务列表
2. **点击录音按钮**：开始录音，界面变为录音状态
3. **长按录音按钮**：连续录音模式
4. **录音完成**：自动调用语音识别和AI拆分
5. **显示结果**：展示拆分后的任务列表

## 🔄 交互流程设计

### 核心流程：语音输入→AI拆分→任务管理

#### 流程图
```
开始
  ↓
打开主页面
  ↓
点击录音按钮
  ↓
录音界面开始录音
  ↓
用户说话（语音输入）
  ↓
完成录音
  ↓
处理界面（语音识别+AI拆分）
  ↓
显示拆分结果
  ↓
用户编辑（可选）
  ↓
保存任务
  ↓
任务列表页面
  ↓
完成
```

#### 异常处理流程
```
录音过程中异常
  ↓
显示错误提示
  ↓
提供重新录音选项
  ↓
返回录音界面
```

#### 网络异常处理
```
API调用失败
  ↓
显示离线模式提示
  ↓
使用本地备用方案
  ↓
保存本地，待网络恢复后同步
```

## 🛠️ 技术架构设计

### 技术选型建议

#### 移动端框架
- **React Native**：推荐
  - 跨平台开发（iOS + Android）
  - 社区活跃，生态丰富
  - 性能表现良好
  - 开发效率高

#### 语音识别
- **科大讯飞SDK**：
  - 实时语音识别SDK
  - 离线语音识别（可选）
  - 高准确率（95%+）
  - 中文优化

#### AI集成
- **百度千帆API**：
  - HTTP REST API调用
  - 请求封装和错误处理
  - 本地缓存机制
  - 网络异常处理

#### 数据存储
- **本地存储**：SQLite数据库
- **云端同步**：Firebase / AWS
- **数据结构**：任务表、设置表、统计表

### 数据结构设计

#### 任务表（tasks）
```sql
CREATE TABLE tasks (
  id INTEGER PRIMARY KEY,
  content TEXT NOT NULL,
  original_text TEXT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  completed_at DATETIME,
  priority INTEGER DEFAULT 0,
  category TEXT,
  ai_generated BOOLEAN DEFAULT 0
);
```

#### 设置表（settings）
```sql
CREATE TABLE settings (
  key TEXT PRIMARY KEY,
  value TEXT,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### API调用流程

#### 语音识别流程
1. **初始化录音**
2. **音频数据收集**
3. **调用科大讯飞SDK**
4. **获取识别结果**
5. **错误处理和重试**

#### AI拆分流程
1. **构造请求参数**
2. **调用百度千帆API**
3. **解析响应结果**
4. **数据本地存储**
5. **用户界面更新**
